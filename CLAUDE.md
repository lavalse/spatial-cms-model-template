# Spatial CMS Community Templates

Community-maintained model templates for Spatial CMS — a spatial data governance control plane.

## Project structure

- `schema/template-v1.json` — JSON Schema (draft-07) for template validation
- `templates/{category}/*.json` — Template files organized by category (general, plateau, osm, gif, municipal)
- `templates/index.json` — Auto-generated catalog (do not edit manually)
- `scripts/validate.js` — Validates all templates against schema, checks duplicate model keys
- `scripts/build-index.js` — Generates templates/index.json from all template files

## Commands

- `npm run validate` — Validate all templates against schema
- `npm run build-index` — Regenerate templates/index.json

## Conventions

- Template files: lowercase-kebab-case.json
- Model keys: lowercase with underscores, prefixed by category (e.g. `plateau_bldg`, `osm_building`, `gif_facility`, `municipal_aed`)
- SRID: 4326 (WGS84) for general/osm/gif/municipal, 6668 (JGD2011) for plateau
- Governance defaults to manual for both approvalMode and publishMode
- `templates/index.json` is auto-generated by CI on merge to main — do not edit manually
- The `dataset` section in templates is a hint only (CMS does not auto-create datasets)
- fieldType `enum_` requires non-empty `enumValues`; `reference` requires `referenceModelKey`
- `referenceModelKey` only resolves within the same template file (no cross-template references)
- Field labels: English for general/plateau/osm; **Japanese** for gif/municipal (matches official spec column names). The schema has no field-level `description` — put English explanations in the model's `description` instead.
